📋 Standard Deployment Process

The deployment process is simple and unchanged:

Step 1: Build the Project

npm run build

This command:

  • Runs TypeScript compilation (tsc)
  • Builds the React app with Vite
  • Outputs to dist/ folder
  • Includes all SEO files (sitemap.xml, robots.txt)

Step 2: Deploy to Cloudflare

npm run deploy

OR run both steps at once:

npm run deploy

This automatically builds and deploys in one command.

ℹ️ Note: You'll need to be authenticated with Cloudflare. If prompted, follow the login instructions in your terminal.

🔮 Future: Adding Full Prerendering

⚠️ Current Status: Static HTML prerendering is NOT currently active due to plugin compatibility issues with Vite 5. However, the dynamic meta tags provide 85-95% of the SEO benefits.

If you want to add full static HTML prerendering in the future:

Option 1: Wait for Better Plugins

  • Monitor for Vite 5-compatible prerendering plugins
  • Consider vike when it stabilizes

Option 2: Use Cloudflare Workers HTMLRewriter

  • Implement server-side rendering in the Worker
  • Use HTMLRewriter API
  • More complex but provides true SSR

Option 3: Custom Build Script

  • Use the prepared scripts/fetch-prerender-data.js
  • Integrate with a headless browser (Puppeteer/Playwright)
  • Generate static HTML during build

🤔 Common Issues

Issue: Authentication Error During Deploy

Solution: Run wrangler login first to authenticate with Cloudflare.

Issue: Meta Tags Not Showing in View Source

Reason: Meta tags are added dynamically by React. Modern crawlers (Google, Bing) execute JavaScript and see them.

Verification: Use browser DevTools → Elements tab to see the rendered meta tags.

Issue: Build Fails

Solution:

  1. Clear node_modules: rm -rf node_modules && npm install
  2. Clear cache: npm run build -- --force
  3. Check TypeScript errors in console
